  3. How do we run the compiled version, and how can we test our program?

    First, copy hw1.i86 to your own directory. You will then need to make it "executable" by typing: 

    $ chmod a+x hw1.i86
    You can now test your program using input and output redirection, combined with the diff command, as follows: 
    $ ./hw1.i86 < s1.in > s1.tgt
$ gcc -Wall hw1.c
$ ./a.out < s1.in > s1.out
$ diff s1.out s1.tgt
$
    The diff command looks for the differences between two files. If the two files are identical diff will return you to the command prompt without producing any output. Otherwise, it will print all the lines where the two files differ. Consider the following example: 
    $ diff s1.out s1.tgt
2,3c2
< 0-0 
< 
---
> 0-0
    This output means that lines 2 and 3 of s1.out are being compared to line 2 of s1.tgt . We see that there is an additional (blank) line in s1.out, and that line 2 is also different in the two files. Sometimes the two lines may look identical on the screen, which suggests there may be a non-printable character or an extra blank space at the end of the line. Before submitting, you should make sure that the output of your program is exactly the same as that of the supplied program.

  4. Can you give us some hints on how to get started?

    Your program should probably have a loop like this: 

      int ch;

  while(( ch = getchar()) != EOF ) { // be careful with parentheses!
     if( ch == 'S' ) {
         ...
     }
     else if( ch == 'A' ) {
         ...
     }
     else if( ch == 'B' ) {
         ...
     }
  }
    [See Course Notes, Chapter 8, slides 4-7 for further explanation.]
    You might like to use an array to store the number of games won by Team A (or Team B) in each of the (up to 5) sets. (Take care not to over-run the array!)

  5. Which team serves first?

    You should assume that Team A always serves for the first game of the match.

  6. Can we use strings?

    Strings are not covered until later in the course and are not recommended for this assignment. However, if you insist on using them, you may find this code useful: 

    char *name[4] = { "Love", "15", "30", "40" };

for( i=0; i < 4; i++ ) {
  printf( "%s", name[i] );
}

  7. When I submit, it says "CPU time limit exceeded"?

    Your program is going into an infinite loop. You can probably fix it by adding code like this: 

    if( ch == EOF) {
   exit(0);
}
    or changing your scanf() statement to this: 
    if( scanf("%c",&ch) <= 0 ) {
   exit(0);
}

  8. How can the score for a set be bigger than 6 ?

    To win a set, one team must score at least 6 games, and at least two games more than the other team. Scores of 5-7 or 7-5 are possible in the first four sets, and higher scores are possible in the fifth set (with games continuing to be played until one team has a two-game lead).

  9. Can you provide sample input and output for the bonus challenge?

    There are a couple of sample tie-break inputs in the bonus directory.

  10. During the tie-break, whose score should be listed first?

    The score for whichever team served the first point of the tie-break should be listed first, not the team serving the current point.
    (Although confusing, this has become standard practice in televised coverage because otherwise the audience would get even more confused with ordering of the score swapping back and forth so frequently.)
